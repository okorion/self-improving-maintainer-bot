# Security Policy

Do not report sensitive security issues in public GitHub issues.

If GitHub private vulnerability reporting is enabled for this repository, use the repository security advisory flow.

If private reporting is not enabled yet, contact the maintainers privately and include:

- A short summary
- Affected version or commit
- Reproduction steps
- Potential impact
- Suggested mitigation, if known

The bot should label security-related issues, but sensitive details should still be handled by humans.
